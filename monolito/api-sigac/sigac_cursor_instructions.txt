SIGAC Backend - Cursor Instructions

==================================================
CONTEXTO GENERAL
==================================================

Quiero que construyas un backend para SIGAC (Sistema Integral de Gestión de Actividades Colaborativas).

Objetivo del sistema:
Permitir autenticación de usuarios, gestión de disponibilidad horaria de colaboradores y coordinación de actividades colaborativas, validando quórum y evitando conflictos de horario.

Arquitectura obligatoria:
- Monolito Modular
- NestJS con TypeScript
- Prisma ORM
- PostgreSQL
- API REST
- JWT para autenticación
- RBAC con roles ADMIN y COLABORADOR
- Swagger/OpenAPI
- Validación con class-validator
- bcrypt para contraseñas
- Logging básico
- Health check endpoint

Reglas obligatorias:
- No microservicios
- No MongoDB
- No Redis
- No BullMQ
- No colas
- No event-driven architecture
- No CQRS
- No WebSockets
- No lógica de negocio en controllers
- No acceso directo a Prisma desde controllers
- Toda lógica debe estar en services
- Acceso a datos encapsulado en repositories
- Arquitectura modular estricta

Módulos:
- auth
- availability
- coordination
- health
- common
- database

Dominio:
- user
- availability
- activity
- activityParticipant
- quorum
- conflict

IMPORTANTE:
No usar términos como tutor, slot, booking o student.
